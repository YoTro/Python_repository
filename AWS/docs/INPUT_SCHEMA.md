# Input File Schemas

This project uses CSV files for both input and output. To ensure tasks run correctly, your input CSV files must contain specific column headers.

## 1. General ASIN Tasks
Most extraction tasks (details, stock, review_count, etc.) require a list of ASINs.

**Required Column:** `ASIN`
**Optional Column:** `URL` (as fallback)

**Affected Tasks:**
- `details`, `stock`, `review_count`, `past_month_sales`, `fulfillment`, `dimensions`, `ranks`, `images`, `videos`, `full_asin_details`.

---

## 2. Keyword Search Tasks
These tasks usually generate the initial ASIN list.

**Required Arguments (via CLI):** `--keyword`
**No Input CSV Required.**

**Affected Tasks:**
- `sales`, `keywords_rank`.

---

## 3. Seller Analysis
Requires Amazon Seller IDs (found in URL as `&seller=XXXX`).

**Required Column:** `ASIN` (The code currently reads from the ASIN column logic in `main.py`).

**Affected Tasks:**
- `feedback`.

---

## 4. Market Intelligence (Analysis)

### `analyze_similarity`
Requires text-heavy product data (ideally generated by the `details` task).

**Required Columns:**
- `ASIN` (for identification)
- `Title` (primary vectorization source)
- `Features` (secondary source, optional)

---

### `analyze_weekly_sales` (Advanced Regression)
Requires historical time-series data of a product's rank and orders.

**Required Columns:**
- `Time`: The date of the record (Format: YYYY-MM-DD).
- `PrimaryRank`: The Best Sellers Rank number on that day.
- `Orders`: The units sold on that day.

**CLI Options to override defaults:**
- `--rank-col`: Specify if your rank column is named differently.
- `--sales-col`: Specify if your sales column is named differently.
- `--date-col`: Specify if your date column is named differently.
